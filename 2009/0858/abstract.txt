Software ICU is already a great tool for monitoring software project's health. However, one significant vital sign is missing, that is the project's "issue". Issue tracking system is now one of the most important and popular tools for task management. And it usually consists of formatted information that it is easy to extract information automatically for sensor data. The use of issue tracking system reflects the task management habit of a software project, which is an essential part to its success.

The project will be introducing issue tracking as a vital sign in software ICU. The components need to be done includes DPD analysis of "issue", several telemetry analysis of issue that showing different aspect of it, and the correct coloring mechanism to demonstrate issue in a meaningful and concise way.
